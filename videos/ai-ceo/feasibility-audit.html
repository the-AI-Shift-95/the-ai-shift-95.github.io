Story Section
Significant
Story claims "access to Slack" as if it's a simple connection
The Experiment section
What the script says
"I gave it tons more data to work with... access to my calendar, access to slack."
Giving Claude access to Slack requires either: (a) Claude Code with a Slack MCP installed, which requires technical setup, or (b) manually copying and pasting Slack messages into the chat. Neither is a simple "connect" like the phrasing implies. This matters because the story is supposed to feel achievable — if it sounds like a 2-click setup but actually requires Claude Code + MCP config, the audience feels misled when they try.
Fix
Reframe in the story: "I started feeding it my data — I'd paste my calendar exports, my Slack threads from the week, drop in my financials." This is honest, still impressive, and sets up Demo 2 correctly (the web interface version where you provide context manually).
Significant
"It would assess my Slack on a daily basis" implies automation
The Experiment section
What the script says
"It would assess my slack on a daily basis, and give me an alarm bell when i was getting sucked back into operational hell."
An automated daily Slack assessment that proactively alerts you is not possible in Claude web or standard Claude Code. It would require a scheduled script that reads Slack via API, sends it to Claude, and pushes a notification back — essentially a custom application. This is buildable with Claude Code but it's not "using Claude," it's building a custom system with Claude as the engine.
Fix
Reframe as a deliberate daily habit, not an automated alert: "Every day I'd open Claude, paste in what I'd done and what my Slack looked like, and it would tell me whether I'd spent time on signal or noise." This is honest and still powerful — the discipline is the point, not the automation.

The editor compositing these panels is a visual trick — not showing a real connection. This is fine for Demo 1 since it's explicitly a simulation. Replace FB Ads panel with something that IS connectable (Stripe, or a manually exported CSV). Alternatively: show FB Ads as a screenshot Sai provides as context (paste in the week's performance data), not as an automatic pull. The visual impact is the same — the difference is the framing. "I drop in my weekly numbers" is honest and still impressive.

Demo 2 — "Here's What You Can Do Right Now" (Live Claude Web)
Critical Context

Demo 2 as written cannot be filmed. It assumes Claude.ai web has live Slack and email integrations — it does not. The entire live demo section needs to be redesigned around what Claude Projects can actually do.

Critical
Slack cannot be "connected" to Claude web interface
Demo 2 Setup + Live Demo
What the script says
"Show the integrations connected: Google Calendar, Slack. Walk through connecting them — it's a few clicks, no code, no technical knowledge."
There is no native Slack integration in Claude.ai Projects. You cannot connect Slack with a few clicks. Slack data cannot be pulled live in Demo 2. This is the most significant blocker in the entire video — it affects both what can be shown in Demo 2 and what the CTA promises to deliver.
Fix
Remove Slack from Demo 2's "connected integrations." The Demo 2 workflow instead is: user provides context via the MD file + pastes relevant info into the chat (or exports a CSV). Frame it as: "You give it context once, then every week you update it. Here's what that looks like." This is honest and still shows value.
Critical
Email cannot be "connected" to Claude web interface
Demo 2 Setup + CTA
What the script says (CTA section)
"It walks you through connecting your calendar, Slack, and email — step by step, no code, no technical knowledge."
No email integration exists in Claude Projects. Gmail, Outlook — none of these can be connected. This promise in the CTA cannot be delivered in the freebie as described.
Fix
Remove email from the CTA promises. The freebie delivers: context file + planning prompts + calendar guidance (if Calendar MCP exists). That's still a strong offer.
Critical
Google Calendar integration in Claude web — needs verification
Demo 2 Setup + Live Demo
What the script says
"The AI pulls the calendar and Slack — show it happening in real time."
Claude.ai Projects has Google Drive integration (confirmed). Whether it includes Google Calendar specifically as a live readable source — not confirmed. Google Drive and Google Calendar are separate products. This needs to be tested on Sai's actual Claude account before scripting the demo around it. If Calendar read is possible, the demo can include it. If not, the approach changes to: user pastes their week's calendar events as text context.
Fix
Sai needs to test this before anything else. Open Claude.ai → create a Project → check if Google Calendar appears as a connectable integration. If yes, test whether it actually reads upcoming events. Report back before Demo 2 is scripted further.
Critical
Claude web cannot CREATE calendar entries — only potentially read them
Demo 2 Live Demo step 5
What the demo guide says
"Show the calendar entries being created with context and links."
Even if Google Calendar integration exists in Claude web for reading, write access (creating calendar events) is not a feature of Claude.ai. Claude generates text output. The user would need to manually create the events from Claude's suggestions. This cannot be shown as "Claude creates the calendar entries" in Demo 2.
Fix
Show Claude outputting the week plan as structured text. Then briefly show the viewer how to copy it to their calendar manually, or note "we include a template that makes this fast." In Demo 1 (simulation), creating calendar entries via Claude Code IS possible — that's the "full version." Demo 2 is the 70% version, and it generates the plan for you to implement.
